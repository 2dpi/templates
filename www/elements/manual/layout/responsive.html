CSS Grid - Responsive

Override columns widths based on viewport size using media queries to achive required results.

To achive good results when the user uses the browser zoom function it's advisable to implement the use of em's instead of px's

http://pxtoem.com/ is a useful tool to help with the conversion. e.g. Browser defaut font size is 16px so to set a breakpoint at 300px = 18.750em

NOTE: Mobile first approach - that is start with smallest view and work your way up. This applies to media query ordering, changing the order will render different results. Each media query applied to an element will only be actioned once.

Popular devices to target are iPhone, iPad, Samsung Galaxy Sx etc refer to http://viewportsizes.com for reference

{% highlight css linenos %} {% raw %} @media screen and (max-width: 320px) { /* smart phone portrait */ .class { width: 100%; } } @media screen and (max-width: 480px) { /* smart phone landscape */ .class { width: 100%; } } @media screen and (max-width: 568px) { /* smart phone landscape (iPhone)*/ .class { width: 100%; } } @media screen and (max-width: 640px) { /* smart phone landscape (samsung S*)*/ .class { width: 100%; } } @media screen and (max-width: 767px) { /* small tablet portrait */ .class { width: 100%; } } @media screen and (max-width: 768px) { /* small tablet portrait */ .class { width: 100%; } } {% endraw %} {% endhighlight %}

NOTES:

Scaling down can be an issue, so consider the following:

  • adding classes manually - scales, static, staggered
  • adding classes dynamically - using javascript

MEDIA QUERY BREAK POINTS (MOBILE > UP)

  • 320 > MOBILE PORTRAIT (ASSUMED BELOW 480)
  • 480 > MOBILE LANSCAPE (BREAK 1)
  • 568 > MOBILE LANSCAPE (BREAK 2)
  • 640 > MOBILE LANDSCAPE (BREAK 3)
  • 768 > IPAD PORTRAIT & DESKTOP (BREAK 3) (exceptions 800/853/966)

STAGGER (ASSUME MAX 5 COLS) 320 > 480 > 586 > 640 > 768

EM GUIDE

NOTE: media queries using EM's are calculated at browser dafault EM = 16px

  • 320@16 20em
  • 480@16 30em
  • 568@16 35.5em
  • 640@16 40em
  • 767@16 47.938em
  • 768@16 48em

NOTE: Ordering plays a part in the output. If your base CSS caters for desktop make sure that consequent media queries go from largest to smallest

  • 768@16 48em
  • 767@16 47.938em
  • 640@16 40em
  • 568@16 35.5em
  • 480@16 30em
  • 320@16 20em
