Style Guide - Spacing

Depending on the context (list, content, structural) spacing will vary around your HTML elements.

A test case would typically be a list / blog view that could have some intro text followed by a leading article, some primary articles and some links.

The issue arises when these are stacked and some contain images etc.

A possible solution:

  • Use orientation (horizontal / vertical)
  • User ordering (first / last)
  • Define alignement (top, right, bottom, left)

Work on the assumption that at some point all elements will be fluss fitting i.e. no margins (in the event a background color is needed) and work back from there allowing for resets and overrides on child items.

the proposed classes:

  • .segement adds padding all round
  • .landscape adds margin top and bottom
  • .portrait adds margin left and right
  • .framed adds margin all round
  • .framed_top adds margin top
  • .framed_right adds margin right
  • .framed_bottom adds margin bottom
  • .framed_left adds margin left
  • .align_top removes padding top
  • .align_right removes padding right
  • .align_bottom removes padding bottom
  • .align_left removes padding left
